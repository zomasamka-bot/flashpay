# Pi Domain Registration Fix - QR Code and Payment Links

## Problem Identified

When customers scanned the QR code generated by FlashPay, they received the error:
```
This domain is not registered yet
```

The QR code was pointing to: `https://flashpay.pi/pay/...` instead of the registered subdomain.

## Root Cause

The app was hardcoding `flashpay.pi` as the domain for QR codes and payment links. However, the actual registered Pi Network subdomain for your app is **`flashpay0734.pi`**.

### Files Affected:
1. **`lib/router.ts`** - QR code URL generation
2. **`lib/config.ts`** - Configuration
3. **`public/.well-known/pi-validation.txt`** - Domain validation

## Changes Applied

### 1. Updated `lib/router.ts`
**Changed:**
- Removed dynamic domain extraction from `config.appUrl` (Vercel URL)
- Added hardcoded Pi Network subdomain constant: `flashpay0734.pi`
- Updated `getPiNetPaymentUrl()` to use the correct subdomain
- Updated `getPiDeepLink()` default domain parameter

**Why:**
- QR codes need to use the Pi-registered subdomain, NOT the Vercel URL
- The Vercel URL (https://flashpay-two.vercel.app) is only for backend APIs
- Pi Browser recognizes only registered Pi Network subdomains (flashpay0734.pi)

### 2. Updated `lib/config.ts`
**Added:**
```typescript
piNetSubdomain: process.env.NEXT_PUBLIC_PI_NET_SUBDOMAIN || "flashpay0734.pi"
```

**Why:**
- Centralizes subdomain configuration
- Allows environment variable override if needed (though default should always be flashpay0734.pi)

### 3. Updated `public/.well-known/pi-validation.txt`
**Changed:**
- Added format: `flashpay0734:YOUR_PI_API_KEY_HERE`
- Added instructions for getting the correct API key

**Why:**
- This file validates ownership of the flashpay0734.pi domain
- Pi Browser checks this file to confirm the domain is properly registered

## Troubleshooting

### Issue: "This domain is not registered yet"
- **Cause**: QR code is still using wrong domain
- **Fix**: Clear browser cache, restart Pi Browser, regenerate QR code
- **Verify**: Check network tab that URL contains `flashpay0734.pi`

### Issue: Links don't open in Pi Browser
- **Cause**: Not using `pi://` protocol
- **Fix**: Verify `getPiNetUrl()` and `getPiDeepLink()` are being called
- **Check**: Look for `pi://` in generated links

### Issue: Domain validation fails
- **Cause**: `/public/.well-known/pi-validation.txt` has wrong format or missing key
- **Fix**: 
  1. Get correct API key from Pi Developer Portal
  2. Format: `flashpay0734:YOUR_KEY_HERE`
  3. Deploy to production
